# Dialog Example — Annotated Session Transcript

This is a compressed example of a full session. Mentor moves are annotated
with `[WHY]` notes to explain the creative choices.

---

## Topic intake

**Mentor:** *"What topic would you like to explore together? Give me a word,
a phrase, or a question — whatever feels right."*

**Brainstormer:** *"Open source sustainability"*

---

## Phase 1 — Planting the 4 roots

**Mentor:** *"Before we start growing ideas, I'm curious: when you think about
open source sustainability, what are the first four lenses or angles that come
to mind? They could be anything — strengths, tensions, dreams, unknowns...
Think of them as the four roots your idea tree will grow from."*

**Brainstormer:**
- A: Funding models
- B: Contributor burnout
- C: Corporate adoption
- D: Community governance

[GOOD: Brainstormer owns the roots. Mentor doesn't suggest or reframe them.]

---

## Round 1

### Mentor adds branches:

```
Root A: Funding models
  ├─ [mentor] Subscriptions create dependency — who holds the switch?
  └─ [mentor] What if the code itself billed for its own maintenance?
```
[WHY A1: logical — funding creates power asymmetry. A2: field jump / personification]

```
Root B: Contributor burnout
  ├─ [mentor] Recognition lags behind contribution by years
  └─ [mentor] A maintainer is a lighthouse keeper — alone, essential, forgotten
```
[WHY B1: logical extension. B2: personification / metaphor — evocative, short]

```
Root C: Corporate adoption
  ├─ [mentor] Companies extract value faster than they return it
  └─ [mentor] What if every npm install came with a shadow invoice?
```
[WHY C1: logical. C2: inversion / constraint gift — provocative question]

```
Root D: Community governance
  ├─ [mentor] Voting systems import politics into technical spaces
  └─ [mentor] Ancient water rights law solved exactly this — look there
```
[WHY D1: logical. D2: historical echo — unexpected but grounded]

**Mentor:** *"Your turn — add whatever wants to grow. You don't have to touch every branch."*

---

### Brainstormer adds (Round 1):

**Brainstormer:**
- A: Grants are one-time, but software is forever
- B: Emotional labour is invisible in commit logs
- D: Benevolent dictators work until they don't

[GOOD: Brainstormer skipped Root C entirely — that is perfectly valid.]

## Round 2

### Mentor adds branches (now growing on brainstormer's branches too):

```
Root A: Funding models
  ├─ [mentor] Grants are one-time, but software is forever
  │   ├─ [mentor] Endowment model: software as cultural heritage
  │   └─ [mentor] What would a Guggenheim for code look like?
  └─ [mentor] What if the code itself billed for its own maintenance?
      ├─ [mentor] Metered OSS: pay-per-execution like serverless
      └─ [mentor] The vending machine that builds itself
```

[NOTE: After Round 1, branches can grow on branches. The tree deepens.]

**Mentor:** *"Your turn — add whatever wants to grow."*

**Brainstormer:** `brainstormed`

## Phase 3 — Harvest (abbreviated)

**Mentor:** *"Beautiful — let's harvest what we've grown."*

### Harvest — Patterns & Insights

1. The word "sustainable" hides two different problems: *money* and *morale* — most solutions only address one.
2. Corporate adoption is treated as the villain but may be the only source of structural funding at scale.
3. Governance models imported from political science tend to recreate the same power failures they were designed to solve.

### Carry-Forward Questions

1. What would an endowment fund for critical OSS infrastructure actually require to launch?
2. Can emotional labour be made legible without bureaucratising it?
3. Who should own the shadow invoice — the project, the foundation, or a new kind of entity?
